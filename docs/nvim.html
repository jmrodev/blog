Preparación y requisitos

Requisitos mínimos para Neovim

Para asegurar que tu experiencia con Neovim y NvChad sea fluida y completa, verifica que cumples con los siguientes requisitos. Estas herramientas son fundamentales para el correcto funcionamiento y la personalización de tu editor.

  • Neovim 0.10.0 o superior: Es el corazón de tu editor.
    • Verifica tu versión: nvim --version
  • Nerd Font: Indispensable para que Neovim y sus plugins muestren iconos y símbolos especiales correctamente. ¡Asegúrate de no usar la variante "Mono" si hay una opción regular, ya que puede causar problemas de alineación!
    • Recomendado: JetBrainsMono Nerd Font (no JetBrainsMonoMono Nerd Font).
  • Ripgrep (opcional, pero altamente recomendado): Esta es una herramienta de búsqueda de texto ultrarrápida que mejora significativamente las capacidades de búsqueda dentro de Neovim.
    • Verifica su instalación: rg --version
  • GCC y Make: Son esenciales si planeas usar plugins de Neovim que requieran compilación (a menudo plugins escritos en C/C++).
    • Verifica su instalación: gcc --version y make --version

¿Cómo verificar los requisitos?

nvim --version
fc-list | grep Nerd
rg --version
gcc --version
make --version

¿Cómo instalar los requisitos?

sudo pacman -S neovim ripgrep gcc make # Manjaro/Arch
sudo apt install neovim ripgrep gcc make # Debian/Ubuntu

En Windows, puedes usar WSL para un entorno Linux completo, o herramientas como Chocolatey/Scoop para instalar ripgrep. Para macOS, Homebrew (brew install ripgrep gcc make) es la forma más sencilla.

Backup y limpieza de instalaciones previas

Si ya usaste Neovim antes, haz un backup y limpia para evitar conflictos:

mkdir ~/backup_nvim
cp -r ~/.config/nvim ~/.local/share/nvim ~/.cache/nvim ~/backup_nvim
rm -rf ~/.config/nvim ~/.local/share/nvim ~/.cache/nvim

Tips

  • Siempre haz backup antes de borrar.
  • Reinicia el terminal tras instalar fuentes o paquetes.
  • Consulta la guía oficial si tienes dudas.

Instalación de Nerd Fonts

¿Por qué usar Nerd Fonts?

Las Nerd Fonts permiten que Neovim y sus plugins muestren iconos y símbolos correctamente, mejorando la experiencia visual y la funcionalidad de herramientas como exploradores de archivos o árboles de sintaxis.

Paso 1: Descargar una Nerd Font

  • Recomendado: JetBrainsMono Nerd Font
  • Descarga el archivo .zip para tu sistema operativo.

Paso 2: Instalar la fuente

Linux

sudo apt install unzip
cd ~/Descargas
unzip JetBrainsMono.zip -d JetBrainsMono
mkdir -p ~/.local/share/fonts
mv ./JetBrainsMono/*.ttf ~/.local/share/fonts
fc-cache -f -v

Windows y macOS

  • Extrae el .zip descargado.
  • Haz doble clic en los archivos .ttf (o .otf si los hay) y pulsa "Instalar".

Paso 3: Configura la fuente en tu terminal

  • Abre la configuración de tu emulador de terminal.
  • Selecciona la fuente JetBrainsMono Nerd Font (o la que hayas elegido).
  • Guarda y reinicia el terminal.

Algunos terminales populares con buen soporte para Nerd Fonts incluyen Alacritty, Kitty, WezTerm (Windows/Linux/macOS), iTerm2 (macOS) y Windows Terminal (Windows).

¿Problemas?

  • Reinicia el terminal o la sesión completa.
  • Verifica que la fuente esté bien escrita y seleccionada en la configuración de tu terminal.
  • Consulta la web oficial de Nerd Fonts para más ayuda.

Instalación de Neovim y NvChad

¿Starter o core?

  • Starter: Es el repositorio recomendado para la mayoría de los usuarios. Proporciona una base robusta y personalizable para tu configuración.
  • NvChad core: Este repositorio es solo para desarrollo o contribución directa al proyecto NvChad. Para personalizar tu editor, siempre debes usar el "starter".

Instalación rápida

git clone [https://github.com/NvChad/starter](https://github.com/NvChad/starter) ~/.config/nvim --depth 1
nvim

¿Qué ocurre al iniciar?

La primera vez que ejecutas nvim después de la clonación, sucederán algunas cosas:

  • Se descargarán e instalarán los plugins y temas básicos de NvChad.
  • Verás una ventana de instalación o progreso; puedes cerrarla con <kbd>q</kbd> cuando el proceso termine.
  • La configuración es mínima y está lista para que empieces a personalizarla.

Si el primer inicio no descarga nada o da errores, asegúrate de tener conexión a internet y verifica los permisos de la carpeta ~/.config/nvim.

Bootstrap explicado

Este es el código principal que inicia NvChad y carga tus plugins. Entenderlo te da una idea de cómo se inicializa tu entorno:

vim.g.base46_cache = vim.fn.stdpath "data" .. "/nvchad/base46/"
vim.g.mapleader = " "

-- bootstrap lazy and all plugins
local lazypath = vim.fn.stdpath "data" .. "/lazy/lazy.nvim"
if not vim.loop.fs_stat(lazypath) then
  local repo = "[https://github.com/folke/lazy.nvim.git](https://github.com/folke/lazy.nvim.git)"
  vim.fn.system { "git", "clone", "--filter=blob:none", repo, "--branch=stable", lazypath }
end
vim.opt.rtp:prepend(lazypath)
local lazy_config = require "configs.lazy"
require("lazy").setup({
  { "NvChad/NvChad", lazy = false, branch = "v2.5", import = "nvchad.plugins" },
  { import = "plugins" },
}, lazy_config)

Una vez instalado, tu directorio de configuración de Neovim tendrá esta estructura:

~/.config/nvim/
├── init.lua

├── LICENSE
├── lua/
│   ├── chadrc.lua
│   ├── configs/
│   ├── mappings.lua
│   ├── options.lua
│   └── plugins/

Tips

  • Si algo falla, puedes eliminar la carpeta ~/.config/nvim y repetir el proceso.
  • Consulta la guía oficial para más ayuda.

Estructura de configuración de NvChad

Carpetas y archivos principales

Comprender la estructura de ~/.config/nvim/ es clave para personalizar NvChad:

~/.config/nvim/
├── init.lua              # Arranque principal y carga de módulos
├── lazy-lock.json        # Estado de plugins y sincronización de Lazy.nvim
├── LICENSE               # Licencia del proyecto
├── lua/
│   ├── chadrc.lua        # Configuración principal: temas, UI, plugins, mappings globales
│   ├── configs/          # Configuraciones avanzadas de plugins, LSP, formatters
│   ├── mappings.lua      # Atajos de teclado personalizados
│   ├── options.lua       # Opciones generales de Neovim
│   └── plugins/          # Plugins adicionales y organización de tus complementos
└── README.md             # Descripción y ayuda sobre tu configuración

¿Qué hace cada archivo?

  • init.lua: Es el punto de entrada de Neovim. Arranca el editor y se encarga de cargar el resto de la configuración.
  • lazy-lock.json: Mantiene un registro del estado de tus plugins, lo que permite a Lazy.nvim sincronizar y gestionar las instalaciones de forma eficiente.
  • LICENSE: La licencia de uso y distribución de NvChad.
  • README.md: Un archivo de información que puedes personalizar para describir tu propia configuración.
  • chadrc.lua: El corazón de tu personalización. Aquí definirás el tema, ajustarás la interfaz de usuario (UI), habilitarás o deshabilitarás plugins predeterminados y configurarás atajos globales.
  • options.lua: Contiene las opciones generales de Neovim (como el número de línea, sangría, etc.) y autocmds (comandos automáticos que se ejecutan bajo ciertas condiciones).
  • mappings.lua: Dedicado a definir tus atajos de teclado personalizados, manteniendo tu configuración organizada.
  • plugins/: Esta carpeta es donde añadirás y organizarás tus plugins adicionales. Puedes dividirlos por propósito (ej: ui.lua para interfaz de usuario, completion.lua para autocompletado, lsp.lua para configuraciones LSP específicas, utils.lua para utilidades varias).
  • configs/: Aquí encontrarás configuraciones más avanzadas para plugins específicos, servidores de lenguaje (LSP), linters y formatters.

¿Plugins o configs?

  • La carpeta plugins/ se usa para declarar y organizar tus plugins adicionales.
  • La carpeta configs/ se usa para configurar opciones avanzadas de cada plugin o sistema (como el LSP).

Tips

  • ¡No edites los archivos del core de NvChad! Realiza todas tus personalizaciones dentro de los archivos de tu carpeta lua/ para evitar perder tus cambios al actualizar NvChad.
  • Haz copias de seguridad de tu configuración, especialmente antes de realizar grandes cambios.
  • Lee los comentarios en cada archivo; suelen contener ejemplos útiles y explicaciones.

Mejores prácticas adicionales

  • Control de versiones: Usa Git para gestionar tu configuración (~/.config/nvim). Esto te permite restaurar versiones anteriores, experimentar con confianza y compartir tu configuración fácilmente.
  • Organización modular: Divide tus configuraciones y plugins en archivos separados y bien nombrados dentro de lua/plugins/ y lua/configs/. Esto facilita el mantenimiento y la personalización a largo plazo.

Starter vs TinyVim

NvChad ofrece dos plantillas principales:

  • Starter: Es la plantilla oficial y recomendada para la mayoría de usuarios. Proporciona una base robusta, fácil de personalizar y mantener, ideal para empezar rápidamente.
  • TinyVim: Es una configuración minimalista y muy ligera. Ideal para quienes buscan la máxima velocidad, simplicidad y control total sobre cada aspecto, construyendo su entorno casi desde cero.

Puedes explorar ambos en los directorios repos/starter/ y repos/tinyvim/ dentro de la instalación de NvChad. Consulta sus README.md respectivos para detalles y ejemplos.

Gestión de plugins (Lazy.nvim)

¿Qué es un plugin?

Un plugin es una extensión que añade funcionalidades a Neovim, como autocompletado inteligente, exploradores de archivos, temas visuales, y mucho más, expandiendo las capacidades del editor base.

¿Qué es Lazy.nvim?

Lazy.nvim es el gestor de plugins moderno que NvChad utiliza por defecto. Permite instalar, actualizar y eliminar plugins de forma eficiente y, a menudo, visual, optimizando el rendimiento de tu editor.

Comandos útiles de Lazy.nvim

  • :Lazy → Abre la interfaz visual de Lazy.nvim, donde puedes gestionar tus plugins, ver su estado y actualizar.
  • :Lazy sync → Instala, actualiza o elimina plugins según los cambios en tu configuración. Ejecuta esto después de modificar tu lista de plugins.
  • :Lazy update → Actualiza todos tus plugins a sus últimas versiones.
  • :Lazy clean → Elimina plugins que ya no están referenciados en tu configuración, liberando espacio.

¿Cómo agregar un plugin?

  1. Edita el archivo nvim/lua/plugins/init.lua (o cualquier otro archivo lua que uses para organizar tus plugins dentro de nvim/lua/plugins/).
  2. Añade una tabla de Lua con la información del plugin. Por ejemplo:
    {
  "ggandor/lightspeed.nvim",
  event = "VimEnter", -- Carga el plugin al iniciar Neovim
},
  3. Guarda el archivo y ejecuta :Lazy sync en Neovim para que se instale el nuevo plugin.

¿Cómo eliminar o desactivar un plugin?

  • Para eliminar un plugin, simplemente borra la tabla correspondiente en tu archivo de configuración (init.lua o el que uses) y luego ejecuta :Lazy sync.
  • Para desactivar temporalmente un plugin, puedes añadir enabled = false a su tabla de configuración:
    {
  "ggandor/lightspeed.nvim",
  enabled = false, -- Desactiva este plugin
},
    Y de nuevo, ejecuta :Lazy sync.

Tips

  • Lee la documentación de cada plugin. Siempre revisa el README.md del repositorio del plugin para conocer sus opciones de configuración avanzadas, dependencias y ejemplos de uso.
  • Usa eventos de carga. Para optimizar el rendimiento de Neovim, carga los plugins solo cuando los necesites utilizando la opción event (ej., event = 'VeryLazy' para plugins que no son críticos al inicio, event = 'InsertEnter' para plugins de autocompletado, o event = 'BufRead' para exploradores de archivos).
  • Haz cambios de uno en uno y prueba que todo funcione antes de añadir más plugins o configuraciones complejas.

Ejemplo avanzado de plugin

Aquí tienes un ejemplo de cómo configurar un plugin que requiere una función config para inicializarse o personalizarse:

return {
  {
    "max397574/better-escape.nvim",
    event = "InsertEnter", -- Carga solo cuando entras en modo inserción
    config = function()
      require("better_escape").setup() -- Llama a la función de setup del plugin
    end,
  },
  -- ...otros plugins
}

Consejos de rendimiento y troubleshooting

  • Si un plugin no funciona como esperas, asegúrate de haber ejecutado :Lazy sync después de tus cambios y revisa que la configuración sea correcta según la documentación del plugin.
  • Para ver si un plugin está causando lentitud en el inicio de Neovim, puedes usar :Lazy profile para analizar los tiempos de carga de cada plugin.

Plugins de la comunidad (nvcommunity)

El repositorio nvcommunity es una valiosa colección de plugins y módulos adicionales desarrollados por la comunidad de NvChad. Estos amplían las capacidades de tu editor con integraciones para autocompletado, LSP, edición, exploradores de archivos y más.

  • Puedes explorar los módulos disponibles en repos/nvcommunity/lua/nvcommunity/.
  • Para usar un módulo, sigue las instrucciones específicas en su README.md correspondiente y añade la configuración sugerida en tu plugins/init.lua (o en el archivo de plugins que organices).
  • Verifica siempre la compatibilidad de cada módulo con tu versión de Neovim y otros plugins antes de integrarlo en tu configuración.

LSP y Mason

¿Qué es LSP?

El Language Server Protocol (LSP) es un estándar abierto que permite a los editores de código (como Neovim) comunicarse con "servidores de lenguaje" externos. Estos servidores proporcionan funciones inteligentes como autocompletado contextual, definiciones de símbolos (salto a definición), referencias, renombrado de variables, diagnósticos de código (errores y advertencias), formateo automático y muchas otras características que transforman tu editor en un IDE completo para cada lenguaje.

¿Qué es Mason?

Mason es un gestor de servidores LSP, linters (analizadores de código estático) y formatters (herramientas de formateo de código). Integrado en Neovim, Mason facilita la instalación y actualización de estas herramientas directamente desde el editor, sin necesidad de manejarlas manualmente en el sistema.

Instalar un servidor LSP

  1. Abre Mason en Neovim ejecutando el comando :Mason.
  2. Busca el servidor de lenguaje que necesitas (ej: pyright para Python, tsserver para TypeScript, rust_analyzer para Rust).
  3. Instálalo directamente desde la interfaz de Mason seleccionándolo y presionando <kbd>i</kbd>.
  4. Después de instalarlo con Mason, debes indicarle a NvChad qué servidores LSP quieres usar. Edita el archivo nvim/lua/configs/lspconfig.lua y añade el nombre del servidor a la tabla servers:
    local servers = { "html", "cssls", "pyright" } -- Añade "pyright" si lo instalaste
  5. Guarda el archivo y reinicia Neovim, o ejecuta :LspRestart para que los cambios tomen efecto.

Ejemplo de configuración para varios lenguajes

Puedes listar múltiples servidores en tu lspconfig.lua para tener soporte en varios lenguajes:

local servers = { "html", "cssls", "tsserver", "pyright", "clangd", "jsonls" }

Comandos útiles y troubleshooting

  • Usa :LspInfo para ver el estado de los servidores LSP activos en tu buffer actual. Te mostrará qué servidor está adjunto y cualquier error.
  • Si el autocompletado o las características del LSP no funcionan, verifica el nombre exacto del servidor y asegúrate de que esté correctamente instalado a través de :Mason y listado en lspconfig.lua.
  • Después de hacer cambios en la configuración del LSP o de instalar un nuevo servidor, siempre reinicia Neovim o ejecuta :LspRestart.
  • Si los diagnósticos (errores/advertencias) no aparecen, asegúrate de que el servidor LSP esté corriendo y de que el buffer del archivo sea reconocido por el LSP (puedes verificarlo con :LspInfo). A veces, guardar el archivo activa los diagnósticos.

Atajos y mappings

¿Qué es un mapping?

Un mapping es un atajo de teclado personalizado que ejecuta un comando o acción específica en Neovim, permitiéndote automatizar tareas y mejorar tu eficiencia.

¿Cómo crear un mapping?

Utilizamos la función vim.keymap.set para crear mappings. Sus argumentos básicos son:

  • Modo: El modo de Neovim en el que el atajo estará activo (ej., "n" para normal, "i" para inserción, {"n", "i"} para ambos).
  • Teclas: La combinación de teclas que activará el atajo (ej., ";").
  • Acción: El comando o función de Lua a ejecutar (ej., ":").
  • Opciones (opcional): Una tabla con opciones adicionales como desc (descripción del atajo, útil para la hoja de trucos).
local map = vim.keymap.set
map("n", ";", ":", { desc = "Entrar al modo comando" })

Ejemplos útiles

Aquí tienes algunos mappings que puedes añadir o modificar en tu nvim/lua/mappings.lua:

  • map("i", "jk", "<ESC>") — Salir del modo inserción a modo normal presionando <kbd>j</kbd> seguido de <kbd>k</kbd>.
  • map({"n","i"}, "<C-s>", ":w<CR>") — Guardar el archivo actual con <kbd>Ctrl</kbd> + <kbd>s</kbd> (funciona en modo normal e inserción).
  • map("n", "ZZ", ":qa<CR>") — Salir de Neovim cerrando todas las ventanas y pestañas con <kbd>ZZ</kbd>.
  • map("n", "<leader>fm", "<cmd>lua vim.lsp.buf.format()<CR>", { desc = "Formatear archivo" }) — Formatear el archivo actual usando el LSP con <kbd>Space</kbd> + <kbd>fm</kbd>.

¿Cómo eliminar un mapping?

Para eliminar un atajo preexistente o uno que hayas creado, usa vim.keymap.del:

vim.keymap.del('n', '<leader>th') -- Elimina el atajo para cambiar tema en modo normal

Tips

  • Consulta :NvCheatsheet o :Telescope keymaps para ver todos los atajos activos en tu configuración y detectar posibles conflictos.
  • Personaliza tus mappings en nvim/lua/mappings.lua para que se adapten a tu flujo de trabajo.
  • Documenta tus atajos con la opción desc para que sean fáciles de identificar y recordar, especialmente cuando uses la hoja de trucos.
  • Haz cambios de uno en uno y prueba que todo funcione correctamente antes de añadir más mappings.

Snippets (LuaSnip)

¿Qué es un snippet?

Un snippet es un fragmento de código reutilizable que puedes insertar rápidamente en tu editor para ahorrar tiempo y evitar errores de escritura repetitiva. Son altamente personalizables y pueden ser dinámicos.

¿Cómo usar LuaSnip?

NvChad utiliza LuaSnip como su motor de snippets. Estos son los atajos básicos para interactuar con ellos:

  • Pulsa <kbd>Tab</kbd> para expandir un snippet (si el cursor está sobre un prefijo de snippet) o para saltar al siguiente campo dentro de un snippet expandido.
  • Pulsa <kbd>Shift</kbd> + <kbd>Tab</kbd> para volver al campo anterior dentro de un snippet.
  • Pulsa <kbd>Ctrl</kbd> + <kbd>l</kbd> para elegir una opción en un campo de elección (si el snippet ofrece múltiples opciones).

¿Cómo crear y cargar snippets?

Puedes crear tus propios snippets en archivos .lua dentro de la carpeta nvim/lua/snippets/. NvChad los cargará automáticamente.

  1. Crea un archivo en nvim/lua/snippets/ con el nombre del lenguaje (ej., lua.lua para snippets de Lua, python.lua para Python).
  2. Dentro de ese archivo, define tus snippets usando la sintaxis de LuaSnip. Ejemplo para Lua (nvim/lua/snippets/lua.lua):
    local ls = require("luasnip") -- Importa la librería LuaSnip
local s = ls.snippet          -- Función para definir un snippet simple
local t = ls.text_node        -- Para texto estático
local i = ls.insert_node      -- Para campos de inserción
local f = ls.function_node    -- Para campos dinámicos (funciones)
local d = ls.dynamic_node     -- Para campos dinámicos
local c = ls.choice_node      -- Para campos con opciones
local rep = require("luasnip.extras").rep -- Para repetir el texto de un nodo

return {
  -- Snippet para una función anónima
  s("fn", {
    t("function("), i(1, "args"), t(")\n  "), i(0), t("\nend")
  }),

  -- Snippet para un bucle for numérico
  s("fori", {
    t("for "), i(1, "i"), t(" = "), i(2, "1"), t(", "), i(3, "10"), t(" do\n  "), i(0), t("\nend")
  }),

  -- Snippet de print con el contenido del campo 1
  s("print", {
    t("print("), i(1, "variable"), t(")")
  }),
}
  3. Guarda el archivo y recarga la configuración de Neovim (o reinicia el editor) para que los nuevos snippets estén disponibles.

Tips

  • Explora friendly-snippets en GitHub. Es una colección masiva de snippets para cientos de lenguajes que puedes usar como inspiración o integrar en tu propia configuración.
  • Puedes crear snippets altamente dinámicos usando funciones de Lua dentro de LuaSnip, lo que permite generar código basado en el contexto o en la entrada del usuario.
  • Lee la documentación oficial de LuaSnip para trucos avanzados y todas las posibilidades que ofrece.

Ejemplo avanzado de snippet

Este snippet de Lua crea un bucle for y permite al usuario definir la variable del bucle, el iterable, y luego posiciona el cursor dentro del cuerpo del bucle:

local ls = require("luasnip")
local s = ls.snippet
local t = ls.text_node
local i = ls.insert_node

return {
  s("for", {
    t("for "), i(1, "var"), t(" in "), i(2, "iterable"), t({" do", "  ", "  "}), i(0), t({"", "end"})
  })
}

Consejos de organización y productividad

  • Organiza tus snippets por lenguaje en subcarpetas dentro de nvim/lua/snippets/ (ej., nvim/lua/snippets/python/, nvim/lua/snippets/javascript/).
  • Adapta y reutiliza snippets existentes de colecciones como friendly-snippets para acelerar tu flujo de trabajo. No necesitas reinventar la rueda.

Temas y personalización visual

¿Cómo cambiar de tema?

NvChad facilita el cambio de temas para personalizar el aspecto de tu editor:

  • Pulsa <kbd>Space</kbd> + <kbd>th</kbd> para abrir el selector de temas integrado.
  • Usa las flechas para navegar por la lista de temas disponibles y pulsa <kbd>Enter</kbd> para aplicar el que te guste.

Personalizar la UI

Puedes establecer tu tema preferido de forma permanente editando el archivo nvim/lua/chadrc.lua:

  • Edita la línea theme = "nombre_del_tema" dentro de la tabla M.ui.
  • También puedes definir una lista de temas para alternar rápidamente con <kbd>Space</kbd> + <kbd>th</kbd>.
M.ui = {
  theme = "onedark", -- Tu tema principal
  theme_toggle = { "onedark", "one_light", "catppuccin" }, -- Temas para alternar
}
  • Guarda el archivo y reinicia Neovim para aplicar los cambios del tema principal.

Tips

  • Explora la lista de temas disponibles en la documentación oficial de NvChad.
  • Puedes crear tus propios temas o modificar los existentes si tienes conocimientos de Lua y del sistema de colores de Neovim.
  • Usa :NvCheatsheet para ver atajos relacionados con la personalización visual y otros comandos útiles.

Personalización avanzada con base46

Los temas en NvChad se gestionan a través del repositorio base46. Esto te permite un control granular sobre los colores de tu editor:

  • Puedes crear o modificar temas editando los archivos en repos/base46/lua/base46/themes/.
  • Para crear un tema propio, copia un archivo de tema existente (ej., onedark.lua), renómbralo y ajusta los valores de color a tu gusto.
  • Una vez creado, tu nuevo tema estará disponible en el selector (<kbd>Space</kbd> + <kbd>th</kbd>) o podrás configurarlo directamente en chadrc.lua.
  • Consulta la documentación en repos/base46/README.md para detalles y ejemplos avanzados sobre cómo definir paletas de colores y estilos.

Teclas principales y atajos en NvChad

Dominar los atajos de teclado es fundamental para ser productivo en Neovim. NvChad viene con un conjunto de atajos predefinidos que puedes personalizar.

Notación de mappings

Para entender los atajos, usa la siguiente notación:

Notación Significado
<C-...> Combinación con la tecla Ctrl
<A-...> Combinación con la tecla Alt

Depuración avanzada con nvim-dap, launch.json y Docker/SSH

La depuración es una parte crucial del desarrollo de software. nvim-dap transforma Neovim en un potente entorno de depuración, similar a la experiencia de un IDE tradicional como VSCode.

Enlaces rápidos

  • ← Volver a atajos y productividad
  • ← Volver al índice principal

¿Qué es nvim-dap?

nvim-dap es el sistema de depuración para Neovim. Implementa el Debug Adapter Protocol (DAP), un protocolo estándar que permite a Neovim comunicarse con "adaptadores de depuración" específicos para cada lenguaje (ej., debugpy para Python, vscode-js-debug para JavaScript).

¿Cómo usar launch.json en Neovim?

nvim-dap puede cargar configuraciones de depuración desde un archivo .vscode/launch.json, el mismo formato que usa VSCode. Esto facilita la migración y el uso de configuraciones existentes.

  1. Crea un archivo .vscode/launch.json en la raíz de tu proyecto.
  2. Define tus configuraciones de depuración dentro de este archivo.
  3. En Neovim, ejecuta el comando para cargar las configuraciones:
    :lua require('dap.ext.vscode').load_launchjs()
  4. Opcional: Puedes mapear este comando a una tecla para mayor comodidad, ejecutándolo antes de iniciar la depuración.

Ejemplo de mappings de teclas para depuración

Definir atajos para las acciones de depuración agiliza mucho el proceso. Puedes añadir estos mappings a tu nvim/lua/mappings.lua:

local dap = require('dap')
local map = vim.keymap.set

-- Iniciar/Continuar depuración
map('n', '<F5>', dap.continue, { desc = 'Iniciar/Continuar depuración' })
-- Pasar por encima (next line)
map('n', '<F10>', dap.step_over, { desc = 'Step Over' })
-- Entrar a función
map('n', '<F11>', dap.step_into, { desc = 'Step Into' })
-- Salir de función
map('n', '<F12>', dap.step_out, { desc = 'Step Out' })
-- Poner/Quitar breakpoint
map('n', '<Leader>b', dap.toggle_breakpoint, { desc = 'Toggle Breakpoint' })
-- Breakpoint condicional
map('n', '<Leader>B', function() dap.set_breakpoint(vim.fn.input('Condición: ')) end, { desc = 'Breakpoint condicional' })
-- Breakpoint de log (logpoint)
map('n', '<Leader>lp', function() dap.set_breakpoint(nil, nil, vim.fn.input('Log: ')) end, { desc = 'Breakpoint de log' })
-- Abrir REPL (Read-Eval-Print Loop) de depuración
map('n', '<Leader>dr', dap.repl.open, { desc = 'Abrir REPL' })
-- Ejecutar la última sesión de depuración
map('n', '<Leader>dl', dap.run_last, { desc = 'Ejecutar última sesión' })
-- Ver variable bajo el cursor
map('n', '<Leader>dv', function() require('dap.ui.variables').hover() end, { desc = 'Ver variable bajo cursor' })
-- Cargar configuraciones de launch.json
map('n', '<Leader>dc', ":lua require('dap.ext.vscode').load_launchjs()<CR>", { desc = 'Cargar launch.json' })

Ejemplos de launch.json por lenguaje

Para Python (usando debugpy)

{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "python",
      "request": "launch",
      "name": "Debug archivo actual",
      "program": "${file}",
      "console": "integratedTerminal",
      "justMyCode": true
    }
  ]
}

Para JavaScript (Node.js, usando vscode-js-debug)

{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug JS (Node.js)",
      "program": "${file}",
      "cwd": "${workspaceFolder}"
    }
  ]
}

Para Java (usando jdtls)

{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "java",
      "request": "launch",
      "name": "Debug Java",
      "mainClass": "com.ejemplo.Main", // Reemplaza con tu clase principal
      "projectName": "MiProyecto"      // Reemplaza con el nombre de tu proyecto
    }
  ]
}

Para C++ (usando cppdbg/GDB)

{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug C++ (GDB)",
      "type": "cppdbg",
      "request": "launch",
      "program": "${workspaceFolder}/a.out", // Ruta a tu ejecutable compilado
      "args": [],
      "stopAtEntry": false,
      "cwd": "${workspaceFolder}",
      "environment": [],
      "externalConsole": false,
      "MIMode": "gdb",
      "setupCommands": [
        { "description": "Enable pretty-printing", "text": "-enable-pretty-printing", "ignoreFailures": true }
      ]
    }
  ]
}

Depuración remota, Docker y Docker Compose

Depurar aplicaciones que se ejecutan en contenedores Docker o en servidores remotos vía SSH es una capacidad poderosa de nvim-dap. El concepto clave es el mapeo de rutas (pathMappings) y la exposición de puertos.

  • Para depurar en Docker o por SSH, asegúrate de que el puerto de depuración (ej: 5678 para Python con debugpy) esté expuesto y sea accesible desde tu máquina local.
  • Usa pathMappings en tu launch.json para que los breakpoints funcionen correctamente. Esto le dice a nvim-dap cómo correlacionar las rutas de archivo locales con las rutas dentro del entorno remoto o del contenedor.
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "python",
      "request": "attach", // Usar "attach" para depuración remota
      "name": "Adjuntar a remoto (Python)",
      "port": 5678,
      "host": "localhost", // Si estás usando un túnel SSH o Docker Desktop
      "pathMappings": [
        {
          "localRoot": "${workspaceFolder}", // Ruta de tu proyecto en la máquina local
          "remoteRoot": "/app"               // Ruta del proyecto dentro del contenedor/servidor remoto
        }
      ]
    }
  ]
}

Tips para Docker Compose

  • Monta el código en la misma ruta: Para simplificar los pathMappings, es una buena práctica montar tu código fuente en la misma ruta dentro del contenedor que en tu máquina local.
  • Exponer el puerto de depuración: Asegúrate de que el servicio de Docker Compose exponga el puerto necesario.
  • Comando de inicio con depurador: Modifica el command del servicio para que inicie tu aplicación con el depurador escuchando en el puerto expuesto.

Ejemplo de servicio en docker-compose.yml para Python:

services:
  app:
    build: .
    volumes:
      - ./:/app # Monta el directorio actual al /app dentro del contenedor
    ports:
      - "5678:5678" # Expone el puerto 5678 del contenedor al 5678 de tu máquina local
    command: ["python", "-m", "debugpy", "--listen", "0.0.0.0:5678", "main.py"] # Inicia la app con debugpy

Tips para SSH

  • Túnel SSH: Para depurar en un servidor remoto, crea un túnel SSH seguro que reenvíe el puerto de depuración del servidor a tu máquina local: 
    ssh -L 5678:localhost:5678 usuario@servidor
  • Verifica el pathMapping: Asegúrate de que 
    remoteRoot
    en tu 
    launch.json
    apunte al path absoluto correcto de tu código en el servidor remoto.

Errores comunes y solución

  • "Breakpoint Unverified": Este es el error más común en depuración remota. Generalmente indica que 
    nvim-dap
    no puede correlacionar la línea de código local con la línea correspondiente en el entorno remoto.
    • Solución: Verifica que tus 
      pathMappings
      sean exactamente correctos y que el código remoto coincida con el local. Pequeñas diferencias en mayúsculas/minúsculas o rutas relativas pueden causar esto.
  • No se conecta al depurador:
    • Solución: Revisa los logs de 
      nvim-dap
      con 
      :DapShowLog
      para ver mensajes de error detallados. Asegúrate de que el puerto esté abierto y que no haya firewalls bloqueando la conexión.

Tips y buenas prácticas

  • Instala los adaptadores DAP: Además de 
    nvim-dap
    , necesitas instalar el adaptador específico para tu lenguaje (ej., 
    nvim-dap-python
    para Python, 
    nvim-dap-js
    para JavaScript). Mason te ayudará con esto.
  • Consulta la documentación de cada adaptador: Los adaptadores pueden tener configuraciones específicas o requisitos de instalación adicionales.
  • Usa mappings: Define tus atajos para iniciar, pausar, poner breakpoints y ver variables fácilmente. Esto acelera drásticamente tu flujo de trabajo de depuración.
  • Siempre carga el 
    launch.json
    : Acostúmbrate a ejecutar 
    :lua require('dap.ext.vscode').load_launchjs()
    antes de iniciar una sesión de depuración que dependa de este archivo.
  • Depuración remota: Asegúrate de que el puerto de depuración esté abierto y accesible desde donde ejecutas Neovim.

Consejos y solución de problemas

¡Felicidades por llegar hasta aquí! Aquí tienes algunos consejos finales y soluciones a problemas comunes para que tu experiencia con Neovim y NvChad sea lo más fluida posible.

Consejos generales

  • Lee la documentación oficial: La documentación de NvChad y Neovim es tu mejor recurso. Contiene información detallada y ejemplos.
  • Haz cambios de uno en uno: Si estás personalizando tu configuración, añade o modifica un elemento a la vez y prueba que funciona. Esto facilita la identificación de la causa si algo falla.
  • Haz copias de seguridad regularmente: Antes de realizar grandes cambios o actualizaciones, haz una copia de seguridad de tu directorio 
    ~/.config/nvim
    .
  • Usa 
    :checkhealth
    : Este comando integrado en Neovim te ayuda a diagnosticar problemas comunes relacionados con dependencias, LSP, y más.
  • Consulta foros y comunidades: Si te atascas, la comunidad de Neovim y NvChad es muy activa. No dudes en buscar ayuda.

Solución de problemas comunes

  • El tema no cambia:
    • Solución: Revisa que el nombre del tema esté bien escrito en 
      chadrc.lua
      (sensible a mayúsculas/minúsculas). Luego, guarda el archivo y reinicia Neovim.
  • Un plugin no funciona o no carga:
    • Solución: Ejecuta 
      :Lazy sync
      para asegurarte de que el plugin esté instalado y actualizado. Verifica su configuración en 
      nvim/lua/plugins/
      y consulta la documentación específica del plugin para posibles dependencias o requisitos.
  • Errores de LSP (autocompletado, diagnósticos):
    • Solución: Primero, verifica que el servidor LSP esté correctamente instalado a través de 
      :Mason
      y que su nombre exacto esté listado en 
      nvim/lua/configs/lspconfig.lua
      . Luego, ejecuta 
      :LspRestart
      o reinicia Neovim. Usa 
      :LspInfo
      para depurar el estado del servidor en el buffer actual.
  • Problemas con atajos de teclado (mappings):
    • Solución: Si un atajo no funciona o hace algo inesperado, usa 
      :NvCheatsheet
      o 
      :Telescope keymaps
      para ver si hay conflictos o si se ha sobrescrito un atajo. Revisa la sintaxis en 
      nvim/lua/mappings.lua
      .
  • Depuración no funciona (nvim-dap):
    • Solución: Revisa los logs de depuración con 
      :DapShowLog
      . Asegúrate de que el adaptador DAP para tu lenguaje esté instalado via Mason. Si es depuración remota, verifica los 
      pathMappings
      en tu 
      launch.json
      y que los puertos estén correctamente expuestos y accesibles.

Tips de productividad

  • Aprende los atajos básicos y personalízalos: Los atajos son la clave de la velocidad en Neovim. Dedica tiempo a aprender los más comunes y personaliza aquellos que usas frecuentemente.
  • Explora nuevos plugins: La comunidad de Neovim es muy innovadora. Mantente al tanto de nuevos plugins que puedan mejorar tu flujo de trabajo.
  • Participa en la comunidad: Contribuye con tus dudas, soluciones o incluso compartiendo tu configuración. Aprender de otros es invaluable.
