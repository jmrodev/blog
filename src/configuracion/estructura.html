Estructura de configuración de NvChad

Carpetas y archivos principales

~/.config/nvim/
├── init.lua              # Arranque y carga de módulos
├── lazy-lock.json        # Estado de plugins y sincronización
├── LICENSE               # Licencia del proyecto
├── lua/
│   ├── chadrc.lua        # Temas, UI, plugins, mappings
│   ├── configs/          # Configs de plugins, LSP, formatters
│   ├── mappings.lua      # Atajos personalizados
│   ├── options.lua       # Opciones generales
│   └── plugins/          # Plugins adicionales y organización
├── README.md             # Descripción y ayuda

¿Qué hace cada archivo?

  • init.lua: Arranca Neovim y carga el resto de la configuración.
  • lazy-lock.json: Guarda el estado de los plugins y permite sincronizar instalaciones.
  • LICENSE: Licencia de uso y distribución.
  • README.md: Información y ayuda sobre tu configuración.
  • chadrc.lua: Cambia el tema, la UI, plugins y atajos globales.
  • options.lua: Opciones generales de Neovim y autocmds.
  • mappings.lua: Define tus atajos de teclado.
  • plugins/: Plugins adicionales, puedes organizar por propósito (ej: utils.lua, editor.lua).
  • configs/: Configuración avanzada de plugins (LSP, formatters, etc).

¿Plugins o configs?

  • plugins/: Añade y organiza plugins por propósito.
  • configs/: Configura opciones avanzadas de cada plugin.

Tips

  • No edites los archivos del core de NvChad, solo los de tu carpeta lua/.
  • Haz copias de seguridad de tu configuración.
  • Lee los comentarios en cada archivo, suelen tener ejemplos útiles.

Mejores prácticas adicionales

  • No edites el core de NvChad: Personaliza solo en tu carpeta lua/ para evitar perder cambios al actualizar.
  • Control de versiones: Usa Git para tu configuración, así puedes restaurar y compartir fácilmente.
  • Organización modular: Divide tus configuraciones y plugins en archivos separados para facilitar el mantenimiento y la personalización.

Starter vs TinyVim

  • starter: Es la plantilla oficial y recomendada para la mayoría de usuarios. Ofrece una base robusta, fácil de personalizar y mantener.
  • tinyvim: Es una configuración minimalista, ideal para quienes buscan velocidad, simplicidad y control total sobre cada aspecto.
  • Elige starter si quieres una experiencia lista para usar y tinyvim si prefieres construir tu entorno desde cero.
  • Ambos se encuentran en repos/starter/ y repos/tinyvim/. Consulta sus README.md para detalles y ejemplos.
